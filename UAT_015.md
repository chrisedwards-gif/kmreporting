# Production fixes 015 — UAT plan

Run this checklist against the Vercel Preview connected to **hos-kitchen-reports-staging**. Do not use production records for destructive or test-only outcomes.

## Release scope

| Area | Expected result |
|---|---|
| Weekly report | A submission panel is visible before Step 1, names every incomplete requirement, saves a draft at any time and enables submission when all six checks pass. |
| Manager 1-1 | The named Kitchen Manager can enter an optional response, acknowledge once, see a success confirmation and see the saved response on the permanent record. Failed attempts retain the typed response. |
| Command search | `Cmd-K` / `Ctrl-K` opens only pages and actions available to the current role. Arrow keys, Enter, Escape and mouse navigation work. |
| Accessibility | Admin, Kitchen Manager and Viewer dashboards plus the weekly-report form have no serious or critical axe violations. |
| Management integrity | Evidence files use the private bucket; RAG overrides require a reason; probation outcomes can be finalised and exported to PDF. |
| Trends and digest | Insight date/site filters retain their selection and management-email test delivery appears in Notifications. |

## Required personas

- One Admin or Group Manager.
- One Kitchen Manager with an active site assignment.
- One Viewer or Finance user.
- Two people should test mobile layout as well as desktop; one tester should use keyboard-only navigation.

## Critical journeys

### 1. Weekly report submission

1. Sign in as a Kitchen Manager and open **Weekly reports → Start weekly report**.
2. Confirm **Submit weekly report** is visible at the top and initially disabled.
3. Enter net sales and aggregate wage cost, then confirm sales, food and labour sources.
4. Confirm the panel changes to **Ready to submit** and the button becomes enabled.
5. Save a draft, refresh, and confirm totals, confirmations, top-up purchases and notes return.
6. Submit and confirm the report opens in its submitted/review state.
7. Repeat with one required confirmation missing; submission must remain blocked and explain why.

### 2. Manager acknowledgement

1. Finalise a 1-1 as Group Manager.
2. Sign in as the named Kitchen Manager and open the delivered review link.
3. Enter a test response and choose **Acknowledge review**.
4. Confirm the success toast/inline message, acknowledged status and saved response.
5. Reopen the URL. The acknowledgement form must not reappear.
6. Confirm a different Kitchen Manager cannot access or acknowledge the review.

### 3. Navigation and access

1. Press `Cmd-K` or `Ctrl-K` for each persona.
2. Search for **weekly**, open the result with the keyboard and return with browser Back.
3. Confirm Kitchen Managers and Viewers cannot discover Administration or group-only pages.
4. In Admin site mode, confirm the site-mode label remains visible and results match the selected kitchen workspace.

### 4. Evidence, judgement and probation

1. Upload an allowed test image and PDF; confirm each opens through an authenticated link.
2. Attempt an unsupported type and an oversized file; both must be rejected.
3. Create a RAG override with a reason, verify calculated and overridden states remain visible, then revoke it with a reason.
4. Save a probation draft, finalise an outcome, and export its PDF. The finalised snapshot must be read-only.

### 5. Trends, digest and reminders

1. Exercise day, month, year and custom trend ranges plus the site filter.
2. Send management digest and reminder tests to the configured UAT override recipient.
3. Confirm Notifications records queued/sent/failed status and the intended recipient without emailing operational users.

## Exit criteria

- No critical journey fails for its authorised persona.
- No cross-site or cross-role record is visible.
- No serious or critical axe violation is introduced.
- Every failure has an issue owner, severity, reproduction steps and screenshot.
- High/critical defects block merge. Medium defects need an agreed owner and target release.
- Product owner and one Kitchen Manager explicitly sign off the release.
